# Venetianam Guide

## 项目简介

`venetianam-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，而是作为一个通用容器，存放各类静态页面资源。

这些页面可能包含信息展示、工具界面、文档索引或其他 Web 内容，每个页面均为独立的 HTML 文件，可直接在浏览器中打开或通过静态托管服务访问。

## 目录说明

```
venetianam-guide/
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：核心目录，所有归档的 HTML 页面均放置于此。每个页面应保持独立，避免跨文件依赖。
- **assets/**：存放页面共用的样式、脚本或图片资源。若页面有独立资源，建议在页面同级目录下放置。
- **index.html**：可选入口文件，用于展示页面列表或导航。

## 页面归档说明

本仓库中的 HTML 页面主要用于以下目的：

- 记录与保存：将分散的 Web 内容以静态页面形式集中归档。
- 演示与测试：作为独立页面供本地预览或功能验证。
- 文档与参考：存放结构化的信息页面，便于查阅。

每个页面应具备以下基本规范：

- 使用有效的 HTML5 语法。
- 内部链接使用相对路径。
- 尽量保持自包含，减少外部依赖。
- 页面标题需明确反映内容主题。

## 维护说明

- **更新**：如需新增或修改页面，请直接提交 Pull Request 或推送到主分支。
- **命名**：页面文件名应简洁、有意义，建议使用小写字母和连字符（例如 `about-info.html`）。
- **清理**：不再需要的页面应及时移除，保持仓库整洁。
- **兼容性**：页面应尽量兼容主流现代浏览器，不强制支持过时版本。
